# 架构图内容分析（按行、列、颜色、方向与层级）

本文档对架构图进行按行/列、颜色、排布方向及层级关系的整理。

---

## 一、整体结构

- **排布**：自上而下 5 个主层 + 右侧 1 个竖向条。
- **主层**：应用层 → 应用框架层 → 系统服务层 → 内核层 → 芯片层。
- **右侧**：IDE和工具链（竖向贯穿全图）。

## 五、层级关系简表

```
应用层
  ├─ 行1：系统应用 | 桌面 | 控制 | 设置 | 电话 | …
  └─ 行2：OpenHarmony SDK | Ability Kit | ArkTS/合辑 | …

应用框架层
  └─ 用户程序框架（虚线父框）
       ├─ UI框架
       └─ 元能力框架

系统服务层
  ├─ 系统基本能力子系统集（左列+右列+底行）
  ├─ 基础软件服务子系统集（纵向）
  ├─ 增强软件服务子系统集（纵向）
  └─ 硬件服务子系统集（纵向）

内核层
  ├─ KAL（内核抽象层）
  ├─ 内核子系统 | Linux Kernel | LiteOS | Hongmeng
  └─ 驱动子系统 → HDF

芯片层
  ├─ 海思平台：L2 | L1 | L0
  └─ 其它芯片平台：L2 | L1 | L0

IDE和工具链（竖向，贯穿全图）
```
