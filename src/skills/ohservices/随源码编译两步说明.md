# 让 SystemAbility 部件随源码编译的两步说明

适用于：新建的 SystemAbility 部件（如 sampletest_service）需要随全量源码一起编译时使用。

---

## 第一步：加入部件白名单

- **文件**：`build/indep_component_whitelist.json`
- **操作**：在文件中任选一个部件名数组（如与 `bluetooth_service` 同组的数组），增加一行：
  ```json
  "你的部件名",
  ```
  例如：`"sampletest_service",`
- **作用**：构建系统只允许编译白名单内的部件；未加入则产品配置里即使写了该部件，也会报错或忽略。

---

## 第二步：加入产品配置

- **文件**：根据目标产品选择继承配置，例如：
  - 标准系统（如 rk3568）：`productdefine/common/inherit/rich.json`
  - 手机：`productdefine/common/inherit/phone.json`
  - 电视：`productdefine/common/inherit/tv.json`
  - 穿戴：`productdefine/common/inherit/wearable.json`
- **位置**：在 JSON 中找到 `"subsystem": "communication"` 的 `components` 数组。
- **操作**：在 `components` 中增加一项（部件名与 bundle.json 中 component.name 一致）：
  ```json
  {
    "component": "sampletest_service",
    "features": []
  }
  ```
- **作用**：使用该继承配置的产品在全量编译时会编译并打包该部件；不配置则不会编入镜像。

---

## 示例：sampletest_service 已完成的修改

1. **indep_component_whitelist.json** 中已存在：
   ```text
   "bluetooth_service",
   "sampletest_service",
   "drivers_peripheral_bluetooth",
   ```

2. **productdefine/common/inherit/rich.json** 的 communication 子系统中已存在：
   ```json
   { "component": "bluetooth_service", "features": [] },
   { "component": "sampletest_service", "features": [] },
   { "component": "ipc", ... },
   ```

因此，使用继承 **rich** 的产品（如 rk3568）全量编译时，**sampletest_service** 会随源码一起编译并打入镜像。

---

## 第三步（若 cfg 中服务使用 uid system/root）：高权限进程白名单

若 init 配置里服务的 **uid** 为 `system` 或 `root`，打包阶段会运行 **process_field_validate.py** 校验，要求该服务必须在产品的**高权限进程白名单**中，否则报错 “some services are not authenticated”。

- **文件**：按产品而定，例如 rk3568 为 `vendor/hihope/rk3568/security_config/high_privilege_process_list.json`。
- **操作**：在 `high_privilege_process_list` 数组中增加一项，**name**、**uid**、**gid** 与 cfg 中一致。例如 sampletest：
  ```json
  { "name": "sampletest", "uid": "system", "gid": ["system", "shell"] }
  ```

sampletest_service 已在 rk3568 的该白名单中配置，全量编译可完整通过。

---

## 新建其他 SA 部件时

若新建另一个 SystemAbility 部件（如 `xxx_service`），重复上述两步即可：

1. 在 `build/indep_component_whitelist.json` 中增加 `"xxx_service"`。
2. 在目标产品的继承配置（如 rich.json）的 communication 的 components 中增加 `{ "component": "xxx_service", "features": [] }`。

---

## 编译镜像

完成上述配置后，在源码根目录（含 `build.sh`）执行：

```bash
./build.sh --product-name rk3568
```

即可全量编译 rk3568 镜像，sampletest_service 等已配置部件会随镜像一起产出。详见本技能 SKILL.md 中「使用 build.sh 编译镜像」一节。

---

## 验证产物（编译完成后）

全量编译成功后，确认以下三项产物存在（路径相对于 `out/rk3568/packages/phone/system/`）：

- `system/lib` 或 `system/lib64/` 下：**libsampletest_server.z.so**
- `system/profile/` 下：**sampletest.json**
- `system/etc/init/` 下：**sampletest.cfg**

在源码根目录执行：

```bash
ls out/rk3568/packages/phone/system/lib/libsampletest_server.z.so \
   out/rk3568/packages/phone/system/lib64/libsampletest_server.z.so 2>/dev/null
ls out/rk3568/packages/phone/system/profile/sampletest.json
ls out/rk3568/packages/phone/system/etc/init/sampletest.cfg
```

三项均存在即表示 sampletest_service 已正确编入镜像。
